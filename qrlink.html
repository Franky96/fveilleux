QR Transfert — Transfert optique de fichiers

← Informatique

À quoi ça sert

QR Transfert permet d'envoyer un fichier d'un appareil à un autre uniquement par caméra et écran, sans Wi-Fi, Bluetooth, câble, ni compte. Les deux appareils n'ont pas besoin d'être sur le même réseau — ils n'ont même pas besoin d'être connectés à quoi que ce soit.

Le fichier est découpé en trames, chaque trame est affichée sous forme de code QR sur l'écran de l'émetteur, et la caméra du récepteur les capte une à une pour reconstruire le fichier original.

Comment ça fonctionne

Le protocole utilise un système d'acquittement stop-and-wait : l'émetteur affiche une trame, le récepteur la confirme par un QR code en retour (ACK), et ainsi de suite jusqu'à la fin du fichier. Trois types de messages circulent :

  • HELLO — nom du fichier, taille, nombre de trames, type MIME
  • DATA — index de trame + données encodées en base64
  • ACK — confirmation de réception renvoyée par le récepteur

Comment l'utiliser

  1. 1Ouvrez l'application sur les deux appareils (téléphone, tablette, ordinateur).
  2. 2Sur l'émetteur : choisissez le mode Envoyer, sélectionnez un fichier.
  3. 3Sur le récepteur : choisissez le mode Recevoir, pointez la caméra vers l'écran de l'émetteur.
  4. 4Le transfert démarre automatiquement trame par trame. L'émetteur attend chaque ACK avant d'avancer.
  5. 5Une fois terminé, le fichier est proposé en téléchargement sur le récepteur.

Lancer l'application

L'application s'ouvre en plein écran pour une meilleure lisibilité des codes QR. Une caméra et HTTPS sont requis.

Limitations

Débit
~1 à 3 Ko/s — inhérent au protocole optique stop-and-wait
HTTPS requis
L'accès caméra (getUserMedia) exige un contexte sécurisé
Caméra obligatoire
Les deux appareils doivent avoir une caméra fonctionnelle
Éclairage
La vitesse varie selon la qualité de l'image et la stabilité des appareils

Le code source de l'application est entièrement lisible dans qr-transfer.html — fichier unique, pas de build, pas de minification.
